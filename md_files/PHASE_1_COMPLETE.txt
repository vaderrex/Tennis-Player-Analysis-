================================================================================
GETTING STARTED
================================================================================

1. Install dependencies:
   pip install -r requirements.txt

2. Create database:
   psql -U postgres -c "CREATE DATABASE tennis_rankings;"

3. Set environment variables:
   $env:SPORTRADAR_API_KEY = "your-key"
   $env:DATABASE_URL = "postgresql+psycopg://user:pass@localhost/tennis_rankings"
   $env:MONGODB_URL = "mongodb://localhost:27017"

4. Run full pipeline:
   python -m tennis_etl.ingestion_pipeline

5. Or run tests:
   python test_ingestion_pipeline.py

6. Verify data:
   psql -U postgres -d tennis_rankings
   SELECT COUNT(*) FROM competitors;

================================================================================
DESIGN DECISIONS
================================================================================

1. MongoDB Staging:
   WHY: Durability, audit trail, decoupling, recovery capability
   HOW: Snapshot storage with idempotent upserts
   BENEFIT: Safe to retry without data duplication

2. SQLAlchemy ORM:
   WHY: Type safety, relationship management, dialect abstraction
   HOW: 6 models with proper FK relationships
   BENEFIT: Easy Phase 2 queries and Phase 3 Streamlit integration

3. Idempotent Architecture:
   WHY: Safe retries, automated failure recovery
   HOW: SportRadar IDs as PKs, fixed MongoDB _id
   BENEFIT: Reliable in production

4. Dialect-Aware Upserts:
   WHY: Different databases have different syntax
   HOW: PostgreSQL (ON CONFLICT), MySQL (ON DUPLICATE), fallback
   BENEFIT: Supports PostgreSQL, MySQL, development databases

5. Optional MongoDB:
   WHY: Flexibility for development iteration
   HOW: SKIP_STAGING environment variable
   BENEFIT: Fast testing, no external dependency required

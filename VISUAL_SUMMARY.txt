================================================================================
PART B: SQL RELATIONAL SCHEMA ✅
================================================================================

Module: tennis_etl/models.py (verified complete)

6 Normalized Tables:

  categories
  ├─ category_id (PK)
  └─ category_name

  competitions
  ├─ competition_id (PK)
  ├─ category_id (FK → categories)
  ├─ parent_id (FK → competitions, self-ref)
  └─ competition_name, type, gender

  complexes
  ├─ complex_id (PK)
  └─ complex_name

  venues
  ├─ venue_id (PK)
  ├─ complex_id (FK → complexes)
  └─ venue_name, city_name, country_name, timezone

  competitors
  ├─ competitor_id (PK)
  └─ name, country, country_code, abbreviation

  competitor_rankings
  ├─ rank_id (PK, auto-increment)
  ├─ competitor_id (FK → competitors, UNIQUE)
  └─ rank, points, movement, competitions_played

Design:
  ✅ SportRadar IDs as primary keys
  ✅ Proper foreign key relationships
  ✅ Self-referential competition hierarchy
  ✅ Unique constraint on rankings
  ✅ Nullable fields for optional data

================================================================================
PART C: MONGODB → SQL TRANSFORM & LOAD ✅
================================================================================

Modules:
  tennis_etl/transforms.py   - Flatten nested structures
  tennis_etl/loader.py       - Load operations
  tennis_etl/database.py     - Connection helpers
  tennis_etl/ingestion_pipeline.py - Orchestration

Transform Pipeline:

  1. extract_from_mongodb()
     └─ Retrieve raw payloads from MongoDB

  2. transform_competitions()
     └─ Extract categories, flatten competition hierarchy
     └─ Output: (categories[], competitions[])

  3. transform_complexes()
     └─ Flatten nested venues array
     └─ Link venues to complex_id
     └─ Output: (complexes[], venues[])

  4. transform_doubles_rankings()
     └─ Iterate ranking groups (ATP, WTA)
     └─ Extract competitors and rankings
     └─ Output: (competitors[], rankings[])

  5. load_all() in foreign-key dependency order:
     ├─ Insert categories
     ├─ Insert competitions (depends: categories)
     ├─ Insert complexes
     ├─ Insert venues (depends: complexes)
     ├─ Insert competitors
     └─ Insert rankings (depends: competitors)

Upsert Strategy:
  PostgreSQL: ON CONFLICT DO UPDATE (dialect-aware)
  MySQL:      ON DUPLICATE KEY UPDATE
  Other:      Lookup + insert/update fallback

QUICK START
================================================================================

1. Install dependencies:
   pip install -r requirements.txt

2. Create database:
   psql -U postgres -c "CREATE DATABASE tennis_rankings;"

3. Set environment variables:
   $env:SPORTRADAR_API_KEY = "your-key"
   $env:DATABASE_URL = "postgresql+psycopg://user:pass@localhost/tennis"
   $env:MONGODB_URL = "mongodb://localhost:27017"

4. Run full pipeline:
   python -m tennis_etl.ingestion_pipeline

5. Run tests:
   python test_ingestion_pipeline.py

Expected output:
  [PHASE 1A] Fetching raw data...
  [PHASE 1B] Staging to MongoDB...
  [PHASE 1C] Transforming and loading...
  Pipeline success!
